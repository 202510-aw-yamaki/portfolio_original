【現象】
- 「一覧へ」のしおり（.bookmark）が、本を閉じた状態（data-state="cover"）だと画面上から消える。
- 本（.book）の表示位置自体は変更したくない。

【考えられる問題点】
1. 親コンテナでクリッピングされている
- `yh_portfolio/css/common.css` の `.stage` が `overflow: hidden;`。
- `yh_portfolio/css/portfolio.css` の `.book` は閉じた状態で `left: 80%` に配置され、しおりは `right: -12px` でさらに右へはみ出す。
- この「はみ出し部分」が `.stage` の右端で切られて見えなくなる。

2. 閉じ状態専用のしおり位置が右外へ寄り過ぎ
- `yh_portfolio/css/portfolio.css` の `.book[data-state="cover"] .bookmark` で `right: -12px`。
- 右端ギリギリ配置のため、画面幅や本の幅により完全に不可視になるケースがある。

3. 閉じ状態と開き状態でしおりの見え方条件が異なる
- 開き状態では本が `left: 50%` に移動するため、しおりがステージ内に残りやすい。
- 閉じ状態だけ `left: 80%` なので、同じしおり設定でも見切れやすい。

【本の位置を変えない修正方法（推奨順）】
A. しおりを「本の外側」ではなく「本の右端の内側」に固定する（推奨）
- 目的: `.book` の位置はそのままに、しおりだけ確実に見えるようにする。
- 例（`yh_portfolio/css/portfolio.css`）:
  - `.bookmark { right: 0; transform: translateX(40%); }`
  - `.book[data-state="cover"] .bookmark { right: 0; transform: translateX(35%); }`
- ポイント: `right` を負値にせず、`transform` 量で“本の右端から少し見える”状態を作る。

B. 閉じ状態だけしおりの突出量を clamp で制限する
- 目的: 画面幅ごとの見切れを抑える。
- 例:
  - `.book[data-state="cover"] .bookmark { transform: translateX(clamp(8px, 2vw, 14px)); right: 0; }`
- ポイント: 小画面では突出を小さくし、最低限の視認性を確保。

C. しおりを cover 側にぶら下げる実装へ変更
- 目的: 本の「右端」を基準にした見え方をより厳密にする。
- 方法:
  - `bookmark` を `.cover` の子要素へ移動し、`.cover` に対して `position: absolute; right: -Xpx;` で制御。
- 注意: DOM構造変更が入るため、最小修正ではない。

【非推奨（今回要件と不一致）】
- `.book` の `left: 80%` を左へ動かす対応（本の位置変更になるため不採用）。
- `.stage { overflow: visible; }` へ変更する対応（他UIのはみ出し管理に影響しやすい）。

【最小修正案】
- `yh_portfolio/css/portfolio.css` の `.book[data-state="cover"] .bookmark` を下記に変更:
  - `right: 0;`
  - `transform: translateX(clamp(8px, 1.6vw, 14px));`
- これで本の座標は固定したまま、右端からしおりを常時視認できる可能性が最も高い。
