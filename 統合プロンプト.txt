# 入力
kspたたき台.html
ksp_keisan.js

# 目的
- 外部JSファイル参照をやめ、JSコードをHTML内へ埋め込み、単一HTMLファイルとして動作させる。

# 統合ルール（厳守）
1) HTMLの構造（doctype/html/head/body、meta、title、link、既存の要素、既存のインラインscript/style）は保持する。勝手に削除・大改変しない。  
2) HTML内に、統合対象のJSを読み込む <script src="..."> がある場合：
   - それを削除し、同じ位置に「インラインの <script>（または type="module" が必要なら <script type="module">）」としてJS本文を埋め込む。
   - defer/async/type/module/crossorigin など属性が付いていた場合は、意味が変わらないように扱う（原則：type="module" の必要性はJS内容から判断。defer/async はインラインでは通常不要なので、タイミングが変わらぬよう埋め込み位置で調整する）。
3) HTML側に統合対象の <script src="..."> が見当たらない場合：
   - </body> の直前にインライン <script> としてJSを挿入する（DOM構築後に動く前提を壊さないため）。
4) JS本文は原則そのまま貼り込む（ラップ関数化、変数名変更、ロジック変更をしない）。ただし、単一HTML化のために必要な最小限の修正は許可：
   - JS内で自分自身のファイル名やURLを前提にしている場合（例：import.meta.url など）や、外部ファイル前提の import / require / fetch(ローカルJSやJSON等) がある場合は検出して、(a) その依存が入力に含まれていない限り単一ファイル化できない旨を「HTML内コメント」に明示し、(b) 可能な範囲で動作を維持する形にする。
   - もし import 文が存在する場合は、まず「単一HTMLにするには当該モジュール本文が必要」である旨をHTMLコメントで列挙し、現時点でできる最善（type="module" を維持しつつ未解決依存を明示）にする。
5) 生成物は「統合済みHTML」1ファイルのみを出力する。説明文や手順は出力しない（必要な注意点はHTMLコメントとしてファイル先頭か該当箇所に残す）。
6) インデントと改行は読みやすく整えるが、HTML/JSの意味が変わる整形はしない。
7) 文字コードはUTF-8想定。HTMLの <meta charset="..."> が存在するなら保持。
